# R&D 计划摘要

> 完整计划见 [RD-PLAN.md](../../dev/RD-PLAN.md)，详细发现见 [findings-summary.md](./findings-summary.md)，项目总览见 [README](../README.md)。

## 项目目标

独立验证 [Percepta](https://percepta.ai/blog/can-llms-be-computers) 的核心主张：标准 transformer 原语（抛物线注意力 + FF 路由）能否实现一台可工作的栈式计算机。

核心结论提前说：**能。编译而非训练（compile, don't train）。**

## 环境

CPU-only，Python 3.12，PyTorch + NumPy，无需 GPU。抛物线编码做内存寻址，hard-max（argmax）做注意力选择，float64 保证数值精度。

## 四个阶段组

**基础原语（1-4）：** 验证凸包查询、抛物线编码、累积和、手工接线栈机。所有原语组合通过。

**训练弯路（5-9）：** 尝试梯度下降学习执行。瓶颈递进：拷贝 → 检索 → 加倍 → **真加法不可学习**。这是重要的否定结论。

**编译突破（11-13）：** 回到编译路线，算术直接写入权重矩阵。100% 正确执行，Forth 等价的通用栈机。

**ISA 扩展（14-20）：** 从 12 操作码扩展到 55 操作码，覆盖算术、比较、位运算、局部变量、堆内存、函数调用、结构化控制流和 WASM 兼容的整数溢出语义。

## 核心结论

Percepta 的主张成立。标准 transformer 原语可以实现可工作的栈式计算机。关键洞察：

1. **编译优于训练**。梯度下降能学会执行结构，但学不会整数加法。编译权重直接绕过这个限制。
2. **训练弯路不是浪费时间**。Phase 5-9 的否定结论（加法不可学习）将我们推回正确的编译路线。
3. **CPU 友好**。不需要 GPU，抛物线注意力 + argmax 就是全部。
